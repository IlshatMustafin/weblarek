# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектурный паттерн приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Абстрактный базовый класс для всех компонентов интерфейса. Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`protected constructor(protected readonly container: HTMLElement)` - принимает и сохраняет корневой DOM-элемент компонента. Инстанцирование напрямую запрещено.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные состояния, автоматически записывает эти данные в свойства экземпляра через `Object.assign` (вызывая соответствующие сеттеры дочерних классов) и возвращает ссылку на DOM-элемент `container`.  
`protected setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>` и безопасной установки картинки с альтернативным текстом.

#### Класс Api
Содержит в себе базовую логику отправки запросов к серверу.

Конструктор:  
`constructor(readonly baseUrl: string, options: RequestInit = {})` - инициализирует базовый URL и прописывает дефолтный заголовок `'Content-Type': 'application/json'`.

Поля класса:  
`readonly baseUrl: string` - базовый адрес сервера.  
`protected options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get<T extends object>(uri: string): Promise<T>` - выполняет типизированный GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер.  
`post<T extends object>(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<T>` - принимает объект с данными для JSON-тела запроса и отправляет их на ендпоинт. По умолчанию выполняется `POST` запрос.  
`protected handleResponse<T>(response: Response): Promise<T>` - защищенный метод, проверяющий ответ сервера на корректность. Возвращает распарсенный JSON-промис или отклоняет его с текстом ошибки.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления. Реализует интерфейс `IEvents`.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<EventName, Set<Subscriber>>` - хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков.

Методы класса:  
`on<T extends object>(eventName: EventName, callback: (event: T) => void): void` - подписка на событие, принимает название события (строку или RegExp) и функцию обработчик.  
`off(eventName: EventName, callback: Subscriber): void` - снятие конкретного обработчика с события.  
`emit<T extends object>(eventName: string, data?: T): void` - инициализация события с передачей данных. Поддерживает маску `*` для глобального прослушивания всех событий.  
`onAll(callback: (event: EmitterEvent) => void): void` - подписка на абсолютно все события приложения.  
`offAll(): void` - полный сброс всей коллекции подписок.  
`trigger<T extends object>(eventName: string, context?: Partial<T>): (data: T) => void` - возвращает фабричную функцию-триггер, генерирующую требуемое в параметрах событие при её вызове.

## Данные

### Интерфейсы данных

#### `IProduct`
Описывает товар в каталоге:
- `id: string` - уникальный идентификатор;
- `description: string` - подробное описание;
- `image: string` - URL изображения;
- `title: string` - название;
- `category: string` - категория;
- `price: number | null` - цена (может быть `null`).

#### `IBuyer`
Данные покупателя для оформления заказа:
- `payment: TPayment | null` - способ оплаты;
- `email: string` - электронная почта;
- `phone: string` - номер телефона;
- `address: string` - адрес доставки.

### Типы данных

#### `TPayment`
Тип, определяющий допустимые способы оплаты:
- `card` - оплата банковской картой;
- `cash` - оплата наличными при получении.

#### `TFormErrors`
Объект, используемый для хранения ошибок валидации данных покупателя. Строго типизирован на основе ключей IBuyer через `Partial<Record<keyof IBuyer, string>>`.

#### `IOrderRequest`
Тип данных запроса на создание заказа. Объединяет IBuyer с информацией о составе корзины через пересечение типов:
- `items: string[]` — массив ID товаров из корзины;
- `total: number` — итоговая сумма заказа.

#### `IOrderResponse`
Ответ сервера на успешное создание заказа:
- `id: string` — уникальный ID созданного заказа;
- `total: number` — подтверждённая сервером итоговая сумма заказа.

### Модели данных

#### Класс `CatalogModel`
Отвечает за хранение и управление товарами в каталоге.

Конструктор:
`constructor(events: IEvents)` - принимает брокера событий.

Поля класса:
`private products: IProduct[]` - хранит массив всех товаров.
`private selectedProduct: IProduct | null` - хранит товар, выбранный для подробного отображения.
`protected events: IEvents` - брокер событий для инициализации процессов перерисовки интерфейса.

Методы класса:
`setProducts(products: IProduct[]): void` - сохранение массива товаров и инициация события `items:changed` с передачей нового массива.
`getProducts(): IProduct[]` - получение массива товаров из модели.
`getProductById(id: string): IProduct | undefined` - поиск и получение одного товара по его id.
`setSelectedProduct(product: IProduct): void` - сохранение товара для подробного отображения и генерация события `card:select` с передачей объекта товара.
`getSelectedProduct(): IProduct | null` - получение товара, выбранного для подробного отображения.

#### Класс `CartModel`
Отвечает за управление корзиной покупок - хранением товаров, выбранных пользователем для покупки.

Конструктор:
`constructor(events: IEvents)` - принимает брокера событий.

Поля класса:
`private items: IProduct[]` - массив товаров, находящихся в корзине.
`protected events: IEvents` - брокер событий для уведомления презентера об изменениях.

Методы класса:
`getItems(): IProduct[]` - получение массива товаров, которые находятся в корзине.
`addItem(product: IProduct): void` - добавление товара в массив корзины и генерация единого события `basket:changed`.
`removeItem(id: string): void` - удаление товара из массива корзины по его id и генерация события `basket:changed`.
`clear(): void` - полностью очищает корзину и генерирует событие `basket:changed`.
`getTotalPrice(): number` - подсчет и получение стоимости всех товаров (товары со значением `price: null` приравниваются к `0`).
`getItemCount(): number` - получение количества товаров в корзине.
`hasItem(id: string): boolean` - проверка наличия товара в корзине по его id.

#### Класс `BuyerModel`
Отвечает за хранение и валидацию данных покупателя, указанных при оформлении заказа.

Конструктор:
`constructor(events: IEvents)` - принимает брокера событий.

Поля класса:
`private payment: TPayment | null` - выбранный способ оплаты.
`private email: string` - электронная почта.
`private phone: string` - номер телефона.
`private address: string` - адрес доставки.
`protected events: IEvents` - брокер событий (внедряется в конструктор, но напрямую методами класса для генерации не используется).

Методы класса:
`setData(data: Partial<IBuyer>): void` - частичное сохранение переданных данных в приватных полях модели без вызова сторонних побочных эффектов.
`getData(): IBuyer` - получение всех данных покупателя в виде единого объекта `IBuyer`.
`clear(): void` - сброс и полная очистка данных покупателя.
`validate(): TFormErrors` - выполняет валидацию всех полей. Возвращает объект с сообщениями об ошибках только для невалидных полей. Если поле валидно, оно отсутствует в результирующем объекте.

## Слой коммуникации  

### Класс ApiService
Класс ApiService реализует слой коммуникации приложения и отвечает за взаимодействие с API сервера «Веб‑ларёк». Он использует композицию с классом Api для выполнения HTTP‑запросов.

Конструктор:  
`constructor(apiInstance: IApi)` - принимает экземпляр класса, реализующего интерфейс IApi, для выполнения запросов.

Поля класса:
`private apiInstance: IApi` — экземпляр для выполнения запросов.

Методы класса:
`getProducts(): Promise<IGetProductsResponse>` - Выполняет GET‑запрос к `/product` для получения списка всех товаров.
`placeOrder(orderData: IOrderRequest): Promise<IOrderResponse>` - Отправляет POST‑запрос к `/order` с данными заказа. Возвращает подтверждение от сервера с ID заказа и итоговой суммой.

### Класс Header
Класс Header отвечает за отображение верхней панели страницы (хедера), включая кнопку корзины и счетчика количества товаров. Наследует базовый класс Component<IHeader>.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - принимает корневой элемент хедера и объект брокера событий.

Поля класса:
`protected counterElement: HTMLElement` - элемент для отображения количества товаров (`.header__basket-counter`).
`protected basketButton: HTMLButtonElement` - кнопка для открытия корзины (`.header__basket`).
`protected events: IEvents` - брокер событий.

Сеттеры:
`set counter(value: number)` - устанавливает текстовое значение внутри counterElement, отображая актуальное количество товаров в корзине.

События: 
При клике на basketButton инициируется событие через брокер: `events.emit('basket:open')`.

### Класс Catalog
Класс отвечающий за отображение каталога карточек на главной странице. Является контейнером, в который динамически подставляются элементы карточек. Наследует базовый класс Component<ICatalog>.

Конструктор:
`constructor(container: HTMLElement)` - принимает корневой элемент-контейнер галереи (`.gallery`).

Поля класса:
`protected catalogContainer: HTMLElement` - ссылка на контейнер внутри галереи (совпадает с container).

Сеттеры:
`set catalog(items: HTMLElement[])` - заменяет всё содержимое контейнера переданным массивом элементов карточек с помощью `replaceChildren`.

### Класс Card
Абстрактный класс, являющийся общим родителем для всех компонентов отображения карточек товара. Наследует Component<ICard>.

Конструктор:
`constructor(container: HTMLElement)` - принимает корневой элемент разметки карточки и находит в нем общие элементы: название (`.card__title`) и цену (`.card__price`) с помощью утилиты `ensureElement`.

Поля класса:
`protected titleElement: HTMLElement` - элемент для отображения названия товара.
`protected priceElement: HTMLElement` - элемент для отображения стоимости товара.

Сеттеры и геттеры:
`set title(value: string)` - устанавливает название товара в textContent.
`get title(): string` - вспомогательный геттер, возвращает текущее имя для передачи в alt картинки дочерних классов.
`set price(value: number | null)` - устанавливает цену товара. Если передано значение null, выводит текст «Бесценно», иначе выводит строку вида `[значение] синапсов`.

### Класс CardCatalog
Класс отвечает за отображение карточки товара в каталоге сетки главной страницы. Наследует Card.

Конструктор:
`constructor(container: HTMLElement, actions?: ICardActions)` - принимает элемент шаблона карточки и необязательный объект действий. При наличии коллбэка вешает слушатель клика на весь контейнер карточки. Полностью изолирован от глобальных событий.

Поля класса:
`protected imageElement: HTMLImageElement` - изображение товара (`.card__image`).
`protected categoryElement: HTMLElement` - категория товара (`.card__category`).

Сеттеры:
`set image(value: string)` - передает ссылку на картинку в метод `setImage`, подставляя `this.title` в атрибут alt.
`set category(value: string)` - устанавливает категорию товара и динамически назначает цветовой CSS-модификатор на основе константы `categoryMap` (при отсутствии совпадений ставит `card__category_other`).

### Класс CardBasket
Класс отвечает за отображение строки элемента товара внутри таблицы корзины. Наследует Card.

Конструктор:
`constructor(container: HTMLElement, actions?: ICardActions)` - принимает элемент шаблона и объект действий. При наличии коллбэка вешает слушатель клика на кнопку удаления. Полностью изолирован от глобальных событий.

Поля класса:
`protected indexElement: HTMLElement` - элемент для отображения порядкового номера товара в списке (`.basket__item-index`).
`protected deleteButton: HTMLButtonElement` - кнопка удаления товара из корзины (`.basket__item-delete`).

Сеттеры:
`set index(value: number)` - устанавливает текстовый порядковый номер товара в корзине.

### Класс CardPreview
Класс отвечает за отображение подробной информации о товаре в модальном окне (превью). Наследует Card.

Конструктор:
`constructor(container: HTMLElement, actions?: ICardActions)` - принимает элемент шаблона полного описания карточки и объект действий. Навешивает коллбэк клика из действий на кнопку `actionButton`. Полностью изолирован от глобальных событий.

Поля класса:
`protected imageElement: HTMLImageElement` - элемент для вывода изображения товара (`.card__image`).
`protected categoryElement: HTMLElement` - элемент для отображения категории товара (`.card__category`).
`protected descriptionElement: HTMLElement` - элемент для вывода подробного описания товара (`.card__text`).
`protected actionButton: HTMLButtonElement` - кнопка для взаимодействия с товаром (`.card__button`).

Сеттеры:
`set image(value: string)` - устанавливает ссылку на картинку, подставляя `this.title` в атрибут alt.
`set category(value: string)` - устанавливает текст категории с динамическим назначением CSS-класса цвета через константу `categoryMap`.
`set description(value: string)` - устанавливает полный текст описания товара в descriptionElement.
`set price(value: number | null)` - переопределяет родительский сеттер. Если цена товара отсутствует (null), выводит текст «Бесценно», принудительно блокирует кнопку действия (`disabled = true`) и меняет её текст на «Недоступно».
`set isInBasket(value: boolean)` - если кнопка имеет текст «Недоступно», метод прерывается. Иначе динамически меняет текст кнопки на «Удалить из корзины» (при true) или «В корзину» (при false).

### Класс Modal
Класс является универсальным контейнером для отображения любого динамического контента (шаблонов корзины, карточки товара в режиме Preview, форм заказа и успешного оформления) в модальном окне. Наследует базовый класс Component<IModalContent>.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - принимает корневой элемент разметки модального окна (`.modal`) и объект брокера событий. Вешает вызов `close()` на кнопку закрытия (`.modal__close`) и на клик по оверлею (`e.target === this.container`). На внутренний `.modal__container` вешает `e.stopPropagation()`.

Поля класса:
`protected closeButton: HTMLButtonElement` - элемент кнопки закрытия модального окна.
`protected contentElement: HTMLElement` - элемент-контейнер (`.modal__content`), внутрь которого вставляется разметка контента.
`protected events: IEvents` - брокер событий.

Сеттеры:
`set content(value: HTMLElement)` - принимает любой готовый DOM-элемент и заменяет им текущее содержимое контента через `replaceChildren`.

Методы:
`open(): void` - добавляет корневому контейнеру класс активности `modal_active` и инициирует событие `events.emit('modal:open')`.
`close(): void` - удаляет у корневого контейнера класс `modal_active`, принудительно очищает внутреннее содержимое через `innerHTML = ''` и генерирует событие `events.emit('modal:close')`.

### Класс Basket
Класс отвечает за отображение содержимого корзины в модальном окне. Он управляет списком добавленных товаров, выводом их суммарной стоимости и блокировкой/разблокировкой кнопки оформления заказа. Наследует базовый класс Component<IBasketView>.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - принимает элемент шаблона корзины и брокер событий. Навешивает клик на кнопку `orderButton` для генерации события.

Поля класса:
`protected basketList: HTMLElement` - элемент-список (`.basket__list`), куда динамически вставляются строки товаров (CardBasket).
`protected totalPriceElement: HTMLElement` - элемент для вывода итоговой стоимости заказа (`.basket__price`).
`protected orderButton: HTMLButtonElement` - кнопка перехода к оформлению заказа (`.basket__button`).
`protected events: IEvents` - брокер событий.

Сеттеры:
`set items(cards: HTMLElement[])` - принимает массив готовых элементов `CardBasket`. Если корзина пуста, динамически создает параграф `<p>` с классом `basket__empty` и текстом «Корзина пуста», заменяет им список и блокирует кнопку `orderButton` через `setAttribute`. Если товары есть — вставляет их элементы и разблокирует кнопку через `removeAttribute`.
`set total(value: number)` - устанавливает значение стоимости товаров в формате `[значение] синапсов` внутри totalPriceElement.

События:
При клике на кнопку orderButton инициируется событие перехода к оформлению заказа: `events.emit('order:open')`.

### Класс Form
Класс инкапсулирует общую логику всех форм в приложении: управление кнопкой отправки, отображение ошибок валидации и сбор данных из полей ввода. Объединяет базовое состояние `IFormState` и уникальный тип полей `T`. Наследует базовый класс Component<IFormState & T>.

Конструктор:
`constructor(container: HTMLFormElement, events: IEvents)` - принимает элемент формы и объект брокера событий. Вешает слушатель `input` на контейнер формы для генерации динамического события `[имя_формы]:[имя_инпута]:change` с передачей вычисляемого объекта `{[fieldName]: value}`. Также вешает слушатель `submit`, предотвращающий дефолтное поведение страницы и вызывающий событие `[имя_формы]:submit`.

Поля класса:
`protected submitButton: HTMLButtonElement` - кнопка отправки формы (`button[type="submit"]`).
`protected errorsElement: HTMLElement` - элемент для вывода текста ошибок валидации (`.form__errors`).
`protected events: IEvents` - брокер событий.

Сеттеры:
`set valid(value: boolean)` - управляет доступностью кнопки отправки (`submitButton.disabled = !value`).
`set errors(value: string[])` - принимает массив строк ошибок и выводит их в `errorsElement`, соединяя через запятую и пробел.

Методы:
`clear(): void` — полностью очищает все поля ввода формы с помощью нативного метода `.reset()`.

### Класс OrderForm
Класс отвечает за выбор способа оплаты и ввод адреса доставки. Наследует класс Form и типизируется интерфейсом IOrderForm.

Конструктор:

Поля класса:
`protected cardButton: HTMLButtonElement` - кнопка выбора оплаты онлайн.
`protected cashButton: HTMLButtonElement` - кнопка выбора оплаты при получении.

Сеттеры:
`set payment(value: string | null)` - сбрасывает класс активности `button_alt-active` с обеих кнопок и подсвечивает модификатором только выбранную. При передаче null сбрасывает обе кнопки.
`set address(value: string)` - динамически находит инпут адреса через querySelector внутри контейнера и устанавливает в него значение.

### Класс ContactsForm

## Презентер

Код презентера (Presenter) располагается в основном управляющем скрипте приложения — `src/main.ts`. Он выступает в роли главного связующего звена между слоем данных (Model) и слоем представления (View), обеспечивая слабую связанность компонентов.

### Подход к реализации
В приложении используется классическая событийно-ориентированная архитектура MVP на базе единой точки входа. Презентер не содержит собственной логики хранения состояний. Взаимодействие со слоем отображения строится на основе инверсии зависимостей (Dependency Inversion): при создании компонентов представления мелких карточек (`CardCatalog`, `CardBasket`, `CardPreview`) и окна `Success` в их конструкторы через объект `actions` внедряются анонимные функции-коллбэки. Это позволяет полностью изолировать компоненты мелких View от моделей данных и глобального брокера событий. Крупные блоки интерфейса общаются с презентером через прямую отправку сигналов в `EventEmitter`.

### Принцип реактивного обновления (Событийная цепочка)
Презентер функционирует исключительно как обработчик событий. В нем строго соблюдается правило реактивности: вызов методов мутации данных в моделях никогда не приводит к синхронному вызову методов `render` у компонентов представления. Обновление интерфейса происходит строго в ответ на события, генерируемые моделями данных (`CatalogModel`, `CartModel`) при изменении их внутреннего состояния, либо при явной генерации сигналов валидации презентером.

## События приложения

### События моделей данных (Model)
- `items:changed` — генерируется `CatalogModel` при получении новых товаров с сервера. Презентер маппит массив данных в массив экземпляров `CardCatalog` с коллбэками открытия превью и рендерит их в `catalogContainer`.
- `basket:changed` — генерируется `CartModel` при добавлении, удалении или очистке корзины. Презентер синхронно обновляет счетчик в `header.counter`, маппит товары корзины в строки `CardBasket` со сквозными индексами (`index + 1`) и рендерит их в `basketView` с пересчитанной итоговой ценой.

### События компонентов представления (View) и Презентера
- `buyer:form-errors` — генерируется Презентером после ручного вызова `buyerModel.validate()`. Передает плоский объект ошибок, на основе которого презентер вычисляет флаги `valid` и списки строк ошибок индивидуально для каждого шага (`OrderForm` и `ContactsForm`), разблокируя кнопки при успехе.
- `basket:open` — клик по иконке корзины в хедере. Презентер рендерит текущую верстку `basketView`, помещает её в `modal.content` и открывает модалку.
- `order:open` — клик по кнопке «Оформить» в корзине. Презентер сбрасывает данные покупателя через `buyerModel.clear()`, рендерит дефолтную пустую форму `orderForm` и открывает её в окне.
- `/^order:.*:change$/` и `/^contacts:.*:change$/` — перехват ввода в текстовые инпуты форм по регулярному выражению. Собранные частичные данные сохраняются в модель `buyerModel.setData(data)`, после чего презентер принудительно запускает `buyerModel.validate()` и генерирует событие ошибок.
- `order:payment-change` — клик по кнопкам оплаты в форме. Данные о выбранном способе сохраняются в `buyerModel.setData`, после чего вызывается валидация и генерация события ошибок.
- `order:submit` — успешная отправка первого шага формы. Презентер рендерит чистую форму `contactsForm` и выводит её на экран в модальном окне.
- `contacts:submit` — отправка формы контактов на втором шаге. Презентер комбинирует данные покупателя из `buyerModel.getData()`, массив строк ID товаров из корзины и итоговую стоимость, отправляя объект на сервер через `apiService.placeOrder`. При успешном ответе:
    - Создается экземпляр `Success`, коллбэк закрытия которого завязан на `modal.close()`.
    - Окно рендерится с суммой ответа сервера и помещается в `modal.content`.
    - Вызываются методы полной очистки моделей: `cartModel.clear()` и `buyerModel.clear()`.
- `modal:open` — активация модального окна. Презентер вешает класс `page__wrapper_locked` на элемент `document.body` для принудительной блокировки скролла страницы.
- `modal:close` — закрытие модального окна. Презентер удаляет класс `page__wrapper_locked` с `document.body`, возвращая стандартную прокрутку страницы.
